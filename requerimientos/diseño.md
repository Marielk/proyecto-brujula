# diseño.md

# Proyecto Brújula --- Guía de Diseño UI/UX

## Versión objetivo: v0.12

------------------------------------------------------------------------

# Objetivo

Esta versión no busca agregar nuevas funciones al motor de simulación.

El objetivo es que **Brújula deje de sentirse como un dashboard** y
comience a sentirse como una experiencia cálida, contemplativa y humana.

La interfaz debe transmitir:

-   esperanza
-   calma
-   claridad
-   curiosidad
-   acompañamiento

Nunca ansiedad.

Nunca urgencia.

Nunca productividad extrema.

------------------------------------------------------------------------

# Principios de diseño

## Brújula es un libro ilustrado interactivo.

No es un software financiero.

No es un dashboard.

No es una planilla.

Cada pantalla debe sentirse como una página de un libro.

------------------------------------------------------------------------

## Mostrar poco.

Explicar bien.

Eliminar ruido visual.

Mucho espacio en blanco.

Respiración.

------------------------------------------------------------------------

## Cada pantalla debe responder una sola pregunta.

Ejemplo:

-   ¿Cuál es mi sueño?
-   ¿Qué caminos existen?
-   ¿Cuál parece mejor?
-   ¿Por qué?
-   ¿Qué puedo hacer hoy?

Nunca mezclar demasiadas ideas.

------------------------------------------------------------------------

# Rediseño del flujo

1.  Pantalla inicial
2.  Definición del sueño
3.  Pantalla de exploración
4.  Comparación de caminos
5.  Camino recomendado
6.  Historia del camino
7.  Carta de Sue
8.  Primer ritual sugerido
9.  Explorar otros caminos

------------------------------------------------------------------------

# Pantalla de carga

Eliminar spinner tradicional.

Mostrar progreso narrativo.

Ejemplo:

🌱 Sue abre el mapa...

✓ Comprendiendo tu sueño.

✓ Recordando tu Perfil de Vida.

✓ Explorando caminos posibles.

✓ Comparando futuros.

✓ Buscando el sendero más amable.

✓ Preparando una carta para ti.

Agregar pequeñas ilustraciones suaves.

Duración mínima visual: 2--3 segundos.

------------------------------------------------------------------------

# Pantalla "Explorando caminos"

Convertirla en protagonista.

Mostrar:

"Encontré varios caminos posibles."

Debajo:

Tarjetas comparables.

Cada tarjeta debe incluir:

-   nombre
-   riesgo
-   esfuerzo
-   tiempo
-   estabilidad
-   pequeño resumen

Solo una destacada.

Las demás aparecen más pequeñas.

------------------------------------------------------------------------

# Camino recomendado

Debe sentirse como una decisión.

No como un resultado.

Agregar:

"Elegimos este camino porque..."

Con tres razones.

Y una advertencia importante.

------------------------------------------------------------------------

# Historia del camino

Eliminar aspecto de lista.

Diseñarla como línea de tiempo ilustrada.

Cada año:

emoji

título

descripción

máximo dos líneas

Ejemplo:

2028

🌱 Primer ahorro importante

Ahora puedes avanzar con menos miedo.

------------------------------------------------------------------------

# Evolución del jardín

Eliminar barras técnicas.

Utilizar elementos naturales.

Ejemplos futuros:

🌱 brote

🌿 planta

🌼 flor

🌳 árbol

Mientras tanto utilizar ilustraciones sencillas.

------------------------------------------------------------------------

# Carta de Sue

Debe parecer una carta.

No un bloque de texto.

Agregar:

-   ilustración pequeña de Sue
-   firma
-   fecha
-   fondo tipo papel

Tipografía ligeramente distinta.

------------------------------------------------------------------------

# Rituales

Transformarlos en tarjetas.

Cada ritual:

icono

duración

beneficio esperado

Ejemplo

☕ Chocolate caliente

15 minutos

Reduce sensación de urgencia.

------------------------------------------------------------------------

# Escenarios alternativos

Nueva sección.

"También encontré estos caminos."

Mostrar 3 alternativas.

Permitir abrirlas.

No recalcular.

------------------------------------------------------------------------

# Comparación visual

Agregar una pantalla tipo mapa.

Tres senderos.

Uno iluminado.

Los otros parcialmente ocultos.

El usuario comprende inmediatamente cuál es la recomendación.

------------------------------------------------------------------------

# Estilo visual

Inspiración:

-   Studio Ghibli
-   Claude Monet
-   Love Nikki
-   Fabulous
-   Jardines botánicos
-   Libros ilustrados
-   Acuarela
-   Pinterest cottagecore

------------------------------------------------------------------------

# Paleta

Mantener tonos pastel.

Verdes suaves.

Lavandas.

Azules grisáceos.

Cremas.

Nunca colores saturados.

------------------------------------------------------------------------

# Animaciones

Muy lentas.

Muy suaves.

Fade.

Parallax ligero.

Movimiento de hojas.

Luz.

Nunca rebotes agresivos.

------------------------------------------------------------------------

# Microinteracciones

Cuando aparece una recomendación:

una hoja cae.

Cuando termina la simulación:

Sue sonríe.

Cuando completas un ritual:

confeti de pétalos.

No usar explosiones visuales.

------------------------------------------------------------------------

# Tipografía

Títulos elegantes.

Texto extremadamente legible.

Mucho interlineado.

------------------------------------------------------------------------

# Navegación

Reducir longitud percibida.

Cada bloque debe poder plegarse.

Agregar navegación flotante:

Resumen

Historia

Carta

Rituales

------------------------------------------------------------------------

# Accesibilidad

Contraste AA.

Modo oscuro.

Texto escalable.

Animaciones opcionales.

------------------------------------------------------------------------

# Estado vacío

Nunca dejar pantallas vacías.

Mostrar mensajes cálidos.

Ejemplo:

"Aún no hemos recorrido este sendero."

------------------------------------------------------------------------

# Estado de error

Nunca mostrar errores técnicos.

Ejemplo:

"Parece que hoy la niebla cubrió el camino. Intentemos nuevamente."

------------------------------------------------------------------------

# Futuras ilustraciones

Reservar espacio para:

Sue

el jardín

las estaciones

los senderos

los árboles

las flores

------------------------------------------------------------------------

# Modo Refugio

Diseño completamente distinto.

No simulación.

Más parecido a un rincón tranquilo.

Colores más cálidos.

Música.

Respiración.

Rituales.

Diario.

------------------------------------------------------------------------

# Consistencia

Todo elemento visual debe responder a la metáfora del viaje y el jardín.

No utilizar componentes con apariencia empresarial.

------------------------------------------------------------------------

# Objetivo de esta versión

Al terminar una simulación el usuario debe sentir:

"No recibí un informe.

Sentí que alguien recorrió muchos caminos por mí y me mostró el que
parece cuidar mejor la vida que quiero construir."

Ese es el criterio principal para evaluar el éxito del rediseño.
